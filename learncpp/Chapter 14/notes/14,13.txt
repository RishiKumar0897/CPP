- temporary/anonymous object: an object that has no name and exists only for a single expression 

#include <iostream>

class IntPair
{
private:
    int m_x{};
    int m_y{};

public:
    IntPair(int x, int y)
        : m_x { x }, m_y { y }
    {}

    int x() const { return m_x; }
    int y() const{ return m_y; }
};

void print(IntPair p)
{
    std::cout << "(" << p.x() << ", " << p.y() << ")\n";
}

int main()
{
    // Case 1: Pass variable
    IntPair p { 3, 4 };
    print(p);

    // Case 2: Construct temporary IntPair and pass to function
    print(IntPair { 5, 6 } );

    // Case 3: Implicitly convert { 7, 8 } to a temporary Intpair and pass to function
    print( { 7, 8 } );

    return 0;
}







- when a function returns by value, the object returned is a temp object 

- a temp class object is an rvalue 
    - can only be used where rvalue expressions are accepted 
- temp objects are created at the point of definition and destroyed at the end of the full expression in which they are defined 

- prefer using static_cast when converting to a fundamental type, and a list-initialized temporary when converting to a class type 
    - static_cast 
        - need to perform a narrowing conversion 
        - want to make it really obvious that we're converting to a type that will result in some different behavior 
        - want to use direct-initialization 
    
    - new object using list-initialization
        - want to use list-initialization (e.g. for protection against narrowing, or because we need to invoke a list constructor)
        - need to provide additional arguments to a constructor to facilitatethe conversion

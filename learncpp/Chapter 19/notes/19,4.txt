- a pointer to a pointer works just like a normal pointer 
    - you can dereference it to retrieve the value pointed to 
    - that value itself is a pointer, so you can dereference it again to get to the underlying value 


int value { 5 };

int* ptr { &value };
std::cout << *ptr << '\n'; // Dereference pointer to int to get int value

int** ptrptr { &ptr };
std::cout << **ptrptr << '\n'; // dereference to get pointer to int, dereference again to get int value




- you cannot set a pointer to a pointer directly to a value 
- but you can set a pointer to a pointer to nullptr


- most common use of pointers to pointers is to dynamically allocated an array of pointers 

int** array { new int*[10] }; // allocate an array of 10 int pointers




- another common use for pointers to pointers is to facilitate dynamically allocated multidimensional arrays 

int** array { new int*[10] }; // allocate an array of 10 int pointers — these are our rows
for (int count { 0 }; count < 10; ++count)
    array[count] = new int[5]; // these are our columns




- deallocating 

for (int count { 0 }; count < 10; ++count)
    delete[] array[count];
delete[] array; // this needs to be done last










- should avoid using pointers to pointers whenever possible

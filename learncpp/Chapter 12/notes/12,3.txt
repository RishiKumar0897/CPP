- a reference is an alias for an existing object 
    - once a reference has been defined, any operation on the referernce is applied to the object being referenced 
    - essentially identical to the object being referenced 


- an lvalue reference acts as an alias for an existing lvalue (such as a variable)

- the type of a reference determines what type of object it can reference 
- lvalue referenec types can be identified by the use of a single & in the type specifier 

// regular types
int        // a normal int type (not an reference)
int&       // an lvalue reference to an int object
double&    // an lvalue reference to a double object
const int& // an lvalue reference to a const int object



- int& is the type of an lvalue reference to an int object, and const int& is the type of an lvalue refernece to a const int object 
- a type that specifies a reference is called a reference type, while the type that can be referenced is called the referenced type 


- 2 kinds of lvalue references 
    1. lvalue reference
    2. const lvalue reference 


- lvalue reference variable: variable that acts as a reference to an lvalue (usually another variable)


#include <iostream>

int main()
{
    int x { 5 };    // x is a normal integer variable
    int& ref { x }; // ref is an lvalue reference variable that can now be used as an alias for variable x

    std::cout << x << '\n';  // print the value of x (5)
    std::cout << ref << '\n'; // print the value of x via ref (5)

    return 0;
}



- you can use a non-const lvalue reference to modify the value of the object being referenced 

#include <iostream>

int main()
{
    int x { 5 }; // normal integer variable
    int& ref { x }; // ref is now an alias for variable x

    std::cout << x << ref << '\n'; // print 55

    x = 6; // x now has value 6

    std::cout << x << ref << '\n'; // prints 66

    ref = 7; // the object being referenced (x) now has value 7

    std::cout << x << ref << '\n'; // prints 77

    return 0;
}




- all references must be initialized 

int main()
{
    int& invalidRef;   // error: references must be initialized

    int x { 5 };
    int& ref { x }; // okay: reference to int is bound to int variable

    return 0;
}




- when a reference is initialized with an object, it is bound to that object 
    - the process by which such a reference is bound is called reference binding 
    - the object being reference is called the referent 


- non const lvalue references can only be bound to a modifiable lvalue 

int main()
{
    int x { 5 };
    int& ref { x };         // okay: non-const lvalue reference bound to a modifiable lvalue

    const int y { 5 };
    int& invalidRef { y };  // invalid: non-const lvalue reference can't bind to a non-modifiable lvalue
    int& invalidRef2 { 0 }; // invalid: non-const lvalue reference can't bind to an rvalue

    return 0;
}




- a reference will usually only bind to an object whose type matches the referenecd type

int main()
{
    int x { 5 };
    int& ref { x };            // okay: referenced type (int) matches type of initializer

    double d { 6.0 };
    int& invalidRef { d };     // invalid: conversion of double to int is narrowing conversion, disallowed by list initialization
    double& invalidRef2 { x }; // invalid: non-const lvalue reference can't bind to rvalue (result of converting x to double)

    return 0;
}



- references cannot be changed to reference another object 

- reference variables follow the same scoping and duration rules that normal variables do 

- the lifetime of a reference and its referent are independent
    - a reference can be destroyed before the object it is referenceing 
    - the object being referenced can be destroyed before the reference 


- dangling references: when an object being referenced is destroyed before a reference to it, the reference is left referencing an object that no longer exists 
    - accessing a dangling reference leads to undefined behavior 

- references are not objects.
    - not required to exist or occupy storage 
    - compiler will try to optimize references away by replacing all occurrences of a reference with the referent

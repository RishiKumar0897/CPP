- use static asser to ensure proper number of array initializers 

#include <array>
#include <iostream>

enum StudentNames
{
    kenny, // 0
    kyle, // 1
    stan, // 2
    butters, // 3
    cartman, // 4
    max_students // 5
};

int main()
{
    constexpr std::array testScores { 78, 94, 66, 77 };

    // Ensure the number of test scores is the same as the number of students
    static_assert(std::size(testScores) == max_students); // compile error: static_assert condition failed

    std::cout << "Cartman got a score of " << testScores[StudentNames::cartman] << '\n';

    return 0;
}


- if we add a new enumerator later but forget to add an initializer to testScores, program won't compile 
- can also use static asser to ensure two different constexpr std:;array have the same length 




- if our enumerators start at 0 and proceed sequentially, we can use an array to hold the name of each enumerator 
    - lets us do 2 things 
        1. index the array using the enumerator's value to get the name of that enumerator 
        2. use a loop to iterate through all of the names, and be able to come back to the enumerator based on the index 
    

- you can't traverse enumerations using a for-each loop 
    - we can create a constexpr std::array containing each of our enumerators and then iterate over that (only works if the enumerators have unique values)

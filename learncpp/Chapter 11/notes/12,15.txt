- std::optional is a class template type that implements an optional value 
- std::optional<T> can either have a value of type T, or not 


- this lets us abandon returning a single value and return two values instead:
    - one bool that indicates whether the function was used 
    - the other of type T that holds the actual return value if the function succeeded or an intermediate value if the function failed 


#include <iostream>
#include <optional> // for std::optional (C++17)

// Our function now optionally returns an int value
std::optional<int> doIntDivision(int x, int y)
{
    if (y == 0)
        return {}; // or return std::nullopt
    return x / y;
}

int main()
{
    std::optional<int> result1 { doIntDivision(20, 5) };
    if (result1) // if the function returned a value
        std::cout << "Result 1: " << *result1 << '\n'; // get the value
    else
        std::cout << "Result 1: failed\n";

    std::optional<int> result2 { doIntDivision(5, 0) };

    if (result2)
        std::cout << "Result 2: " << *result2 << '\n';
    else
        std::cout << "Result 2: failed\n";

    return 0;
}





- you can construct a std::optional<T> either with or without a value 


std::optional<int> o1 { 5 };            // initialize with a value
std::optional<int> o2 {};               // initialize with no value
std::optional<int> o3 { std::nullopt }; // initialize with no value




- to see if an std::optional has a value, choose one of the following 

if (o1.has_value()) // call has_value() to check if o1 has a value
if (o2)             // use implicit conversion to bool to check if o2 has a value




- to get the value from a std::optional, choose one of the following 

std::cout << *o1;             // dereference to get value stored in o1 (undefined behavior if o1 does not have a value)
std::cout << o2.value();      // call value() to get value stored in o2 (throws std::bad_optional_access exception if o2 does not have a value)
std::cout << o3.value_or(42); // call value_or() to get value stored in o3 (or value `42` if o3 doesn't have a value)





- std::optional has a usage sytnax that is essentially identical to that of a pointer 


- pros and cons 

pros 
    - effectively documents that a function may or may not return a value 
    -  don't have to remember which value is being returned as a sentinel 
    - syntax for using std::optional is convenient and intuitive 

cons 
    - have to make sure that the std::optional contains a value before getting the value 
        - if you dereference a std:optional that does not contain a value, we get undefined behavior 
    - does not provide a way to pass back info about why the function failed 


- best practice is to use std::optional for functions that may fail unless the function needs to return additional information about why it failed 

- you can also use std::optional as an optional function parameter 


#include <iostream>
#include <optional>

void printIDNumber(std::optional<const int> id = std::nullopt)
{
    if (id)
        std::cout << "Your ID number is " << *id << ".\n";
    else
        std::cout << "Your ID number is not known.\n";
}

int main()
{
    printIDNumber(); // we don't know the user's ID yet

    int userid { 34 };
    printIDNumber(userid); // we know the user's ID now

    printIDNumber(62); // we can also pass an rvalue

    return 0;
}





- advantages of this approach
    1. documents that the parameter is optional 
    2. can pass in an rvalue 


- problematic if T is an expensive-to-copy type 
    - only use if T would normally be passed by value, otherwise use const T* 



- prefer std::optional for optional return types 
- prefer function overloading for optional function parameters (when possible) or use std::optional if T would normally be passed by value 
    - favor const T* if T is expensive to copy

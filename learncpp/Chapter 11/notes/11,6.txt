- template: describes what a function or class looks like
    - can use one or more placeholder types, which represent some type that is not known at the time the template is defined, but will be provided later 
    - once template is defined, compiler can use it to generate as many overloaded functions or classes as needed, using actual types 


- compiler can use a single template to generate a family of related functions/classes, each using a different set of actual types 

- because the actual types aren't determined until the template is used in a program, the author of the template doesn't have to try to anticipate all of the actual types that might be used 
- template code can be used with types that didn't even exist when the template was written, which makes it flexible and future-proof 


- function template: function-like definition that is used to generate one or more overloaded functions, each with a different set of actual types 

- initial function template used to generate other functions is called the primary template 
- functions generated from the primary template are called instantiated functions 

- when creating the primary function template, we used placeholder types for parameters, return types, or types used in the function body that we want to be specified later by the user of the template 




template <typename T> // this is the template parameter declaration defining T as a type template parameter
T max(T x, T y) // this is the function template definition for max<T>
{
    return (x < y) ? y : x;
}



- best practice: use a single capital letter starting with T to name type template parameters that are used in trivial or obvious ways and represent "any reasonable type"
    - if non-obvious, then use a more descriptive name

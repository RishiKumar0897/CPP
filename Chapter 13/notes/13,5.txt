- similar to function overloading, C++ also supports operator overloading 
    - lets us define overloads of existing operators so we can make those operators work with program-defined data types 


- basic operator overloading
    - define a function using the name of the operator as the function's name 
    - add a parameter of the appropriate type for each operand in left-to-right order 
        - one of these params must be a class type or enum 
    - set return type to whatever type makes sense 
    - use return statement to return the result of the operation 


- when compiler encounters the operator in an expression and one or more of the operands is a class/enum type, compiler will check to see if there is an overloaded operator function that it can use to resolve the call 


- we can overload the ""<<"" to output an enumerator 

#include <iostream>
#include <string_view>

enum Color
{
	black,
	red,
	blue,
};

constexpr std::string_view getColorName(Color color)
{
    switch (color)
    {
    case black: return "black";
    case red:   return "red";
    case blue:  return "blue";
    default:    return "???";
    }
}

// Teach operator<< how to print a Color
// std::ostream is the type of std::cout, std::cerr, etc...
// The return type and parameter type are references (to prevent copies from being made)
std::ostream& operator<<(std::ostream& out, Color color)
{
    out << getColorName(color); // print our color's name to whatever output stream out
    return out;                 // operator<< conventionally returns its left operand

    // The above can be condensed to the following single line:
    // return out << getColorName(color)
}

int main()
{
	Color shirt{ blue };
	std::cout << "Your shirt is " << shirt << '\n'; // it works!

	return 0;
}






- name of the function is operator<< 
- it has 2 params
    - left parameter is output stream, has type std::ostream
        - use pass by reference here bc we don't want to make a copy of std::ostream object when function is called 
    - right parameter is Color object 

- << returns its left operand, so the return type matches the type of left operand which is std::ostream&
-out << getColorName(color) fetches color's name as a std::string_view ant prints it to the output stream 




- teaching operator>> how to input an enumeration works similarly 

#include <iostream>
#include <limits>
#include <optional>
#include <string>
#include <string_view>

enum Pet
{
    cat,   // 0
    dog,   // 1
    pig,   // 2
    whale, // 3
};

constexpr std::string_view getPetName(Pet pet)
{
    switch (pet)
    {
    case cat:   return "cat";
    case dog:   return "dog";
    case pig:   return "pig";
    case whale: return "whale";
    default:    return "???";
    }
}

constexpr std::optional<Pet> getPetFromString(std::string_view sv)
{
    if (sv == "cat")   return cat;
    if (sv == "dog")   return dog;
    if (sv == "pig")   return pig;
    if (sv == "whale") return whale;

    return {};
}

// pet is an in/out parameter
std::istream& operator>>(std::istream& in, Pet& pet)
{
    std::string s{};
    in >> s; // get input string from user

    std::optional<Pet> match { getPetFromString(s) };
    if (match) // if we found a match
    {
        pet = *match; // dereference std::optional to get matching enumerator
        return in;
    }

    // We didn't find a match, so input must have been invalid
    // so we will set input stream to fail state
    in.setstate(std::ios_base::failbit);

    // On an extraction failure, operator>> zero-initializes fundamental types
    // Uncomment the following line to make this operator do the same thing
    // pet = {};

    return in;
}

int main()
{
    std::cout << "Enter a pet: cat, dog, pig, or whale: ";
    Pet pet{};
    std::cin >> pet;

    if (std::cin) // if we found a match
        std::cout << "You chose: " << getPetName(pet) << '\n';
    else
    {
        std::cin.clear(); // reset the input stream to good
        std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
        std::cout << "Your pet was not valid\n";
    }

    return 0;
}









- key differences:
    - std::cin has type std::istream, so std::istream& is the return type and left parameter type 
    - pet parameter is a non-const reference, allows >> operator to modify the value of the right operand that is passed in if there is a match during extraction
